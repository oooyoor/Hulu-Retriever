# 多线程测试说明

## 概述

已修改代码以支持通过命令行参数传入线程数，并创建了自动化测试脚本，可以按顺序执行所有测试并对比不同线程数的结果。

## 修改内容

### 1. C++ 文件修改

以下三个文件已支持可选的线程数命令行参数（第4个参数）：
- `HnswOffsetIO.cpp` - Raw NVMe 测试
- `HnswFsIO.cpp` - Single Layer FS 测试  
- `HnswMultiLayerFsIO.cpp` - Multi Layer FS 测试

**新的命令行格式：**
```bash
./execs/offset <config_json_path> <dataset_name> [repeat_id] [num_threads]
./execs/single <config_json_path> <dataset_name> [repeat_id] [num_threads]
./execs/multi <config_json_path> <dataset_name> [repeat_id] [num_threads]
```

如果提供了 `num_threads` 参数，会覆盖配置文件中的 `num_threads` 值。

### 2. 脚本修改

#### `run_test.sh`
- 支持可选的第二个参数（线程数）
- 用法：`./run_test.sh <exec_name> [num_threads]`

#### `raw_nvme_experiment.sh`
- 支持可选的第一个参数（线程数）
- 用法：`./raw_nvme_experiment.sh [num_threads]`

### 3. 新增脚本

#### `run_all_tests.sh` - 主测试脚本
按顺序执行所有测试，支持多线程数对比。

**用法：**
```bash
sudo ./run_all_tests.sh [线程数列表，用空格分隔]
```

**默认线程数：** 2 4 8 16

**执行顺序：**
1. Raw NVMe 测试（对所有线程数）
2. Single Layer FS 测试（对所有线程数）
3. Multi Layer FS 测试（对所有线程数）

**示例：**
```bash
# 使用默认线程数 (2, 4, 8, 16)
sudo ./run_all_tests.sh

# 自定义线程数
sudo ./run_all_tests.sh 2 4 8 16 32
```

#### `run_all_tests_background.sh` - 后台运行脚本
在后台运行所有测试，输出日志到文件。

**用法：**
```bash
sudo ./run_all_tests_background.sh [线程数列表]
```

**功能：**
- 自动生成带时间戳的日志文件
- 保存进程 PID 到 `test_run.pid`
- 提供监控命令提示

**示例：**
```bash
# 后台运行测试
sudo ./run_all_tests_background.sh 2 4 8 16

# 查看日志
tail -f test_run_*.log

# 检查进程状态
ps aux | grep run_all_tests
```

## 测试流程

### 手动执行（按顺序）

1. **Raw NVMe 测试**
   ```bash
   sudo ./raw_nvme_experiment.sh [线程数]
   # 或
   sudo ./run_test.sh offset [线程数]
   ```

2. **Single Layer FS 测试**
   ```bash
   sudo ./run_test.sh single [线程数]
   ```

3. **Multi Layer FS 测试**
   ```bash
   sudo ./run_test.sh multi [线程数]
   ```

### 自动化执行

**前台运行：**
```bash
cd /home/zqf/Hulu-Retriever/MakeFsWorkSpace
sudo ./run_all_tests.sh 2 4 8 16
```

**后台运行：**
```bash
cd /home/zqf/Hulu-Retriever/MakeFsWorkSpace
sudo ./run_all_tests_background.sh 2 4 8 16
```

## 结果位置

测试结果会按线程数自动组织到不同目录：

- **Raw 结果：** `./end_results/raw_results/Offset_results/<dataset>/<threads>_<ef>_<io_depth>/<query_cnt>/<repeat_id>/`
- **Single 结果：** `./results/Fs_results/<dataset>/<threads>_<ef>_<io_depth>/<query_cnt>/<repeat_id>/`
- **Multi 结果：** `./results/MultiLayer_Fs_results/<dataset>/<threads>_<ef>_<io_depth>/<query_cnt>/<repeat_id>/`

目录名中的 `<threads>` 会反映实际使用的线程数。

## 注意事项

1. **配置文件备份：** `run_all_tests.sh` 会自动备份和恢复 `Config.json`，临时修改线程数进行测试。

2. **磁盘准备：** Raw 测试的第一个线程数会执行完整的磁盘准备流程，后续线程数测试会跳过磁盘准备以节省时间。

3. **测试间隔：** 每个测试之间会有 10 秒的间隔，避免系统过载。

4. **权限要求：** 所有测试脚本都需要 `sudo` 权限。

5. **重新编译：** 修改 C++ 文件后，需要重新编译：
   ```bash
   cd /home/zqf/Hulu-Retriever/MakeFsWorkSpace
   ./build_clean.sh offset
   ./build_clean.sh single
   ./build_clean.sh multi
   ```

## 示例：完整测试流程

```bash
# 1. 进入工作目录

# 2. 重新编译（如果修改了代码）
```
